# Gefest Telegram Bot

Telegram-бот для ЖК та комерційних об'єктів компанії «Гефест» (https://gefest.ua/).
Бот парсить каталог приміщень із сайту, зберігає його в локальній базі SQLite
і дозволяє користувачу переглядати квартири/офіси з фото, плануванням та цінами
прямо в Telegram.

## Структура проєкту

| Файл | Призначення |
|---|---|
| `config.py` | Налаштування: токен бота, URL сайту, інтервал оновлення, контакти |
| `scraper.py` | Парсер сайту gefest.ua (requests + BeautifulSoup) |
| `database.py` | Робота з SQLite (таблиця `flats`) |
| `bot.py` | Логіка Telegram-бота (aiogram 3.x): меню, списки, картки приміщень |
| `main.py` | Точка входу: ініціалізація БД, планувальник оновлень, запуск бота |
| `gefest.db` | Файл бази SQLite (створюється автоматично) |

## Як працює парсер

1. **Список приміщень.** Парсер відкриває сторінку загального пошуку
   `https://gefest.ua/search`, проходить по всіх сторінках пагінації
   (`?page=2`, `?page=3`, ...) і для кожної картки квартири/офісу витягує:
   - ID приміщення (`flatid-XXXXX` з посилання),
   - назву ЖК/об'єкта та його "слаг" (наприклад `ellada`, `platon`),
   - посилання на сторінку приміщення,
   - кількість кімнат / тип ("1-кімнатна", "Студія", "Офіс" тощо),
   - секцію, номер, поверх, площу, ціну, статус ("У продажу" / "Продано"),
   - превʼю-фото квартири.

2. **Деталі приміщення.** Для кожного знайденого приміщення парсер додатково
   відкриває його персональну сторінку (`.../flatid-XXXXX`) і дотягує:
   - чергу будівництва ("Черга": напр. "Еллада-6"),
   - точний тип приміщення ("Студія", "2-кімнатна", "Офіс" і т.д.),
   - посилання на завантаження плану квартири — це конструюється як
     `<URL приміщення>/planimage` (саме на цей URL веде кнопка
     "Завантажити план" на сайті, і він віддає готове JPG-зображення).

3. **Збереження.** Усі зібрані приміщення повністю перезаписують вміст таблиці
   `flats` у SQLite (`database.replace_all_flats`). Це гарантує, що продані
   квартири зникають зі списку, а нові — з'являються, без ручного очищення.

Парсер навмисно зроблений "ввічливим": між запитами витримується невелика
пауза (`REQUEST_DELAY = 0.4` сек у `scraper.py`), щоб не перевантажувати сайт.
Повний обхід (~166 приміщень + їх детальні сторінки) займає кілька хвилин.

## Як часто приходять оновлення

- Оновлення каталогу запускається:
  - **одразу при старті бота**, якщо база даних порожня (перше наповнення);
  - **далі — за розкладом, кожні `PARSE_INTERVAL_HOURS` годин** (за замовчуванням
    **12 годин**, налаштовується в `config.py`).
- За оновлення відповідає `APScheduler` (`AsyncIOScheduler`) у `main.py`:
  фонове завдання `update_catalog` виконується в окремому потоці
  (`asyncio.to_thread`), щоб не блокувати обробку повідомлень бота під час
  тривалого парсингу.
- Кожне оновлення повністю перезаписує таблицю `flats`, тому інформація в
  боті завжди відповідає поточному стану сайту на момент останнього обходу
  (з затримкою максимум у `PARSE_INTERVAL_HOURS` годин).

## Як працює бот (сценарій користувача)

1. `/start` — користувач бачить привітання, загальну кількість об'єктів і
   приміщень у продажу та обирає категорію:
   - 🏠 **Квартири** — житлові приміщення (студії, 1-, 2-, 3-кімнатні тощо);
   - 🏢 **Офіси та комерційні приміщення**.

   Поділ на категорії відбувається **за реальним типом приміщення**
   (поле "Тип квартири" з картки на сайті), а не за назвою ЖК — оскільки
   деякі об'єкти ("ЖК «Кімолос»", "ЖК «Родос»") фактично продають лише офіси,
   а "Апарт-комплекс «Ітака»" — лише квартири.

2. Після вибору категорії бот показує список об'єктів (ЖК/БЦ), де є
   приміщення цього типу, із кількістю доступних варіантів у дужках.

3. Після вибору об'єкта бот надсилає приміщення сторінками по
   `FLATS_PER_PAGE` (за замовчуванням 5): кожне — окремим повідомленням
   із превʼю-фото, коротким описом (номер, тип, поверх, площа, ціна, статус)
   і кнопкою **«Детальніше»**. Внизу сторінки — кнопки навігації
   («« Назад» / «Далі »») та повернення до списку об'єктів.

4. Натиснувши **«Детальніше»**, користувач отримує:
   - альбом із двома зображеннями — фото-превʼю квартири та план планування
     (те саме зображення, що завантажується кнопкою "Завантажити план"
     на сайті);
   - повну картку з інформацією: ЖК, черга, секція, номер, поверх, тип,
     площа, ціна, статус, посилання на сторінку приміщення на сайті;
   - кнопки **«Написати в Telegram»** (`@romanova_neruhomist`) та
     **«Подзвонити»** (`+380930828903`) для зв'язку з менеджером;
   - кнопку **«‹ До списку квартир»** для повернення до списку приміщень
     цього об'єкта.

## Запуск

1. Встановити залежності: `pip install -r requirements.txt`
2. Створити файл `.env` на основі `.env.example` і вписати `BOT_TOKEN`
   (отримати у `@BotFather`).
3. Запустити: `python main.py`

При першому запуску бот одразу наповнює базу (перший обхід каталогу), а далі
оновлює її автоматично кожні 12 годин, поки працює.
